## **安装配置证书软件**

```bash
wget https://github.com/OpenVPN/easy-rsa/releases/download/v3.1.7/EasyRSA-3.1.7.tgz
tar -zxvf EasyRSA-3.1.7.tgz 
cd EasyRSA-3.1.7/
vi vars
set_var EASYRSA_REQ_COUNTRY     "US"                        #国家
set_var EASYRSA_REQ_PROVINCE    "California"                #省
set_var EASYRSA_REQ_CITY        "Shanghai"                  #城市
set_var EASYRSA_REQ_ORG "Copyleft Certificate Co"           #组织
set_var EASYRSA_REQ_EMAIL       "test@example.net"          #邮箱
set_var EASYRSA_REQ_OU          "My Organizational Unit"    #公司、组织
```

## **创建证书**

```bash
./easyrsa init-pki   			   #1、初始化，在当前目录创建PKI目录，用于存储整数
./easyrsa build-ca  			   #2、创建根证书，会提示设置密码，用于ca对之后生成的server和client证书签名时使用，其他提示内容直接回车即可
./easyrsa gen-req server nopass    #3、创建server端证书和私钥文件，nopass表示不加密私钥文件，提示内容直接回车即可
./easyrsa sign server server       #4、给server端证书签名，提示内容需要输入yes和创建ca根证书时候的密码
./easyrsa gen-dh   				   #5、创建Diffie-Hellman文件，密钥交换时的Diffie-Hellman算法
./easyrsa gen-req client nopass    #6、创建client端的证书和私钥文件，nopass表示不加密私钥文件，提示内容直接回车即可
./easyrsa sign client client       #7、给client端证书前面，提示内容输入yes和创建ca根证书时候的密码
#./easyrsa gen-req homeServer nopass 
#./easyrsa sign client homeServer  
openvpn --genkey secret ta.key     #8、生成 ta.key 文件 这一步是可选操作，生成的ta.key主要用于防御DoS、UDP淹没等恶意攻击。

tree   							   #检查是否有ca根证书、客户端服务端证书、客户端服务端私钥
.
```

## **安装openvpn并写入服务端配置文件**

```bash
yum -y install openvpn
# sudo apt install openssl        # oepnssl
# sudo apt install libssl-dev
# sudo apt install liblzo2-2      # lzo
# sudo apt install liblzo2-dev
# sudo apt install libpam0g-dev   # pam

vim /etc/openvpn/server.conf
----------------------------------------------------------------------
port 11058 
proto tcp
dev tun
ca /etc/openvpn/ssl/ca.crt
cert /etc/openvpn/ssl/server.crt
key /etc/openvpn/ssl/server.key 
dh /etc/openvpn/ssl/dh.pem
#给客户端分配的地址池
server 10.8.0.0 255.255.255.0
#允许客户端访问的内网网段
;push "route 172.16.1.0 255.255.255.0"
#地址池记录文件位置，未来让openvpn客户端固定ip地址使用的
ifconfig-pool-persist ipp.txt
#允许客户端与客户端之间通信
client-to-client
#存活时间，10秒ping一次，120秒如果未收到响应则视为断线
keepalive 30 120
max-clients 10
#日志位置，记录openvpn状态
status /etc/openvpn/openvpn-status.log
#openvpn日志记录位置
log /var/log/openvpn.log 
# 日志等级
verb 3
persist-key
persist-tun
#客户端密钥（证书和私钥）是否可以重复
duplicate-cn
#启动lzo数据压缩格式
comp-lzo

```

```bash
sudo iptables -t nat -A POSTROUTING -s 10.8.0.0/24 -o eth0 -j MASQUERADE
sudo echo 1 > /proc/sys/net/ipv4/ip_forward
sudo sysctl -p
```

第一条命令允许客户端使用服务器作为Internet代理，它将客户端流量（网络10.8.0.x）呼由一服务器上的eth0网络接口上。eth0接口通常是广域网接口。  
第二条命令是允许内核转转IPv4流量，这对于VPN来说是必需的。

## **启动并检查端口**

```bash
systemctl start openvpn@server
systemctl enable openvpn@server

# 手动启动
#openvpn --daemon --cd /etc/openvpn --config client.ovpn --log-append /var/log/openvpn.log

# --daemon：openvpn以daemon方式启动。
# --cd dir：配置文件的目录，openvpn初始化前，先切换到此目录。
# --config file：客户端配置文件的路径。
# --log-append file：日志文件路径，如果文件不存在会自动创建。
```

如果发现客户端与服务端之间的传输速率非常低，可以在服务器配置中增加缓冲区：（这一点非常重要）

```bash
sndbuf 393216
rcvbuf 393216
push "sndbuf 393216"
push "rcvbuf 393216"
```

## **OpenVPN客户端配置（linux端）**

**配置openvpn**

```bash
yum -y install openvpn
vim /etc/openvpn/client.conf
-----------------------------------------------------------------
client
dev tun
proto tcp
remote xxxxxx 11058
resolv-retry infinite
nobind
ca /etc/openvpn/ssl/ca.crt
cert /etc/openvpn/ssl/client.crt
key /etc/openvpn/ssl/client.key.
remote-cert-tls server
verb 3
persist-key
persist-tun
comp-lzo
status /etc/openvpn/openvpn-status.log
log /var/log/openvpn.log 

# 将前面生成的客户端证书拿过来【ca.crt  client.key  client.crt】
home:/etc/openvpn# tree /etc/openvpn/ssl/
/etc/openvpn/ssl/
├── ca.crt
├── client.crt
└── client.key

1 directory, 3 files
 
# 启动
systemctl start openvpn@client
systemctl enable openvpn@client
```
