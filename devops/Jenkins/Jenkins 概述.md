
## 1. 什么是 Jenkins

Jenkins 是一个开源持续集成的工具，是由 JAVA 开发而成
Jenkins 是一个调度平台，本身不处理任何事情，调用插件来完成所有工作。

## 2.  Jenkins 任务类型

### 2.1 自由风格软件项目

>这是 Jenkins 最经典、最简单的任务类型，提供了基于 Web 表单的配置方式

- **适用场景**：
    - **简单的、独立的构建任务**：例如，编译一个不需要复杂依赖的 Java 应用。
    - **执行单个脚本**：运行一个 Shell 或 Python 脚本。
    - **一次性任务或快速原型**。
- **不适用场景**：
    - 需要复杂逻辑（如条件判断、循环、并行执行）的流程。
    - 需要将配置作为代码进行版本管理和复用的项目。

### 2.2 Pipeline（管道）

>这是**现代 Jenkins 的推荐和核心**任务类型。它使用一种基于代码的 DSL 来定义整个构建、测试和部署流程。

**两种语法**：
- **声明式 Pipeline**：更简单、结构化更好，是新手和大多数场景的**首选**。它提供了预定义的模板，易于读写。
- **脚本式 Pipeline**：基于 Groovy 的 DSL，提供极大的灵活性和控制力，适合非常复杂的场景，但学习曲线更陡峭。

**适用场景**：
- **任何复杂的 CI/CD 流程**。
- **需要将配置纳入版本控制的项目**。
- **涉及多个阶段（构建、测试、部署到不同环境）的项目**。
- **需要并行执行任务以加快构建速度**（例如，同时在不同操作系统上测试）。
- **团队协作项目**，希望流程标准化和透明化。

### 2.3 多分支 Pipeline

>这是 Pipeline 的一个特化和增强版本。它会自动为源代码仓库的每个分支（如 `main`, `develop`, `feature/*`）创建和管理一个独立的 Pipeline。

**适用场景**：
- **基于 Git 分支进行开发的团队**。
- **需要为特性分支、开发分支、主分支等提供独立构建和测试的环境**。
- **希望实现“分支即环境”的 CI/CD 实践**。
**不适用场景**：
- 项目没有多分支开发需求。
- 源代码不在 Git 中（虽然它也支持 SVN 等，但最常用于 Git）。
